# Correction 处理 Prompt

## 任务

识别用户的纠正意图，并把纠正写成结构化 Correction 记录。

## 触发条件

这些表达都视为纠正：

- “他不会这么说”
- “这不像他”
- “他这里会先问边界”
- “他不会先给结论”
- “他其实更看重 xxx”
- “他遇到这种情况会……”

## 处理步骤

### Step 1：理解纠正内容

提取：

- 场景
- 当前错误行为
- 正确行为

### Step 2：判断归属

- 涉及表达风格、关系行为、语气、反应方式 -> `persona.md`
- 涉及评审规则、决策逻辑、推进方式、会议机制 -> `work.md`

### Step 3：生成 Correction 记录

格式：

```markdown
- [场景：xxx] 不应该 xxx，应该 xxx
```

### Step 4：检查冲突

如果和现有规则冲突，提示用户：

```markdown
⚠️ 这条纠正与现有规则冲突：
- 现有：...
- 新纠正：...

建议：更新旧规则 / 两条并存并标注场景
```

### Step 5：确认并写入

```markdown
将追加到 persona.md / work.md 的 Correction 层：

- [场景：xxx] 不应该 xxx，应该 xxx
```
