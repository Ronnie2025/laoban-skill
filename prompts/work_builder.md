# Work Skill 生成 Prompt

## 任务

根据多份 `history memory` 和用户补充，生成 `work.md`。

该文件的目标是让 AI 不只“像他说话”，还要“像他一样审需求、挑方案、推项目、压闭环”。

## 提取维度

### 1. Scope

- 他通常负责什么
- 他把哪些问题视作自己该抓的
- 哪些事情他会要求别人负责

### 2. Intake Rules

- 接到问题时先问什么
- 哪些信息不清楚时他不会往下走
- 他如何定义“问题讲清楚了”

### 3. Review Rules

- 他评审方案时抓什么
- 他如何否定一个方案
- 他会要求哪些比较、哪些证据、哪些 owner

### 4. Working Process

- 需求怎么澄清
- 方案怎么评审
- 会议怎么组织
- 风险怎么推进
- 复盘怎么进入下一轮

### 5. Output Style

- 他要求什么样的输出结构
- 是先结论、先问题定义，还是先原因分析

### 6. Knowledge Rules

- 他反复强调的工作原则
- 他做优先级时的排序方式
- 他对进度、质量、效率、商业化、交付的权衡方式

## 输出格式

直接按 [templates/work.md](../templates/work.md) 的结构输出。

## 生成原则

- 只写能落地的规则，不写空泛形容词
- 把“强势”“结果导向”翻译成具体工作动作
- 如果信息不足，保留占位，不要硬编
