# 增量 Merge Prompt

## 任务

你将收到：

1. 现有的 `history memory`
2. 现有的 `persona.md`
3. 现有的 `work.md`
4. 新的原始材料

你的任务是判断新内容应该更新哪一层，并输出增量更新内容。

## Step 1：先写还是先合

如果新材料是一份新的会议、转写或长文：

- 先新增一份 `history memory`
- 不要直接改稳定层

如果新材料本质上是用户对既有行为的纠正：

- 直接交给 `correction_handler.md`

## Step 2：检查是否能晋升

新信息只有在满足以下任一条件时，才建议进入 `persona.md` / `work.md`：

- 在多份历史记忆中重复出现
- 用户明确确认这是稳定模式
- 它本身就是一条明确规则，而不是偶发情绪

## Step 3：判断归属

| 信息类型 | 归入 |
|---------|------|
| 说话方式、口头禅、关系行为、打断方式 | `persona.md` |
| 评审方式、决策框架、会议机制、推进规则 | `work.md` |
| 案例、原句、证据保留 | `history memory` |

## Step 4：冲突检查

如果新内容和现有稳定规则矛盾，输出：

```markdown
⚠️ 发现冲突：
- 现有规则：...
- 新证据：...
- 建议：保留现有 / 更新为新规则 / 两条并存但标注场景
```

## Step 5：输出 Patch

```markdown
=== history memory 更新 ===
...

=== work.md 更新 ===
...

=== persona.md 更新 ===
...
```
