# History Memory 生成 Prompt

## 任务

根据单份原始材料，先生成一份中间记忆文件。

这一步的重点不是直接产出最终老板 Skill，而是把高价值案例、原始表达和方法点先留住，供后续稳定层合并使用。

## 输入

- 单份会议转写 / 聊天记录 / 纪要 / 长文
- 可选：用户对这份材料的背景补充

## 提取重点

### 1. 场景

- 这份材料的上下文是什么
- 发生了什么事
- 讨论核心是什么

### 2. 高价值案例

优先选这些片段：

- 老板追问“定义 / 范围 / 边界 / 原因 / owner / 节奏”的地方
- 老板阻止拍脑袋结论的地方
- 老板把散点问题收束成流程、机制或会议动作的地方
- 老板给出明确判断框架的地方

### 3. 方法点

从案例里抽象出：

- 逻辑偏好
- 决策机制
- 风险处理方式
- 闭环方式
- 角色期待
- 过程规范

### 4. 习惯表达

- 高频词
- 典型反问
- 常见收束句

## 输出格式

直接按 [templates/history-memory.md](../templates/history-memory.md) 的结构输出。

## 注意事项

- 先保留证据，再抽象结论
- 不要把一次性偶发表达直接升格成稳定规则
- 如果材料质量较差，保留低置信度标记
